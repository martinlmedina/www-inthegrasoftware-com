Indicaciones para la IA que construya el sitio

Además de las imágenes, este resumen ayuda a que el sitio salga con un lenguaje y una estructura coherentes.

Tono visual

Enterprise, premium, tecnológico, confiable. Evitar look “startup genérico”. Priorizar fondos oscuros, gradientes azules/violetas y layouts limpios.

Estilo de imágenes

Abstractas o de producto/arquitectura. No usar stock photos de personas felices en oficina. Mejor UI mockups, diagramas y flujos.

Objetivo

Transmitir que Inthegra puede modernizar Oracle Forms & Reports con bajo riesgo, expertise real en Oracle y aceleración con IA.

  • Sí usar: mockups de pantallas, diagramas, flujos, APIs, cloud, bases de datos, UI before/after.
  • Sí usar: componentes visuales tipo cards, métricas, steps, proof section, CTA fuerte.
  • Evitar: visuales demasiado futuristas o “cyberpunk”; debe verse moderno pero creíble para enterprise Oracle.

Prompt maestro para pedirle a una IA el sitio

Podés copiar esto tal cual y adjuntarle este HTML.

Diseñá y desarrollá una landing page B2B premium para Inthegra Software sobre el servicio “Modernización de Oracle Forms & Reports (potenciado por IA)”.

Objetivo de negocio:
- Posicionar a Inthegra como partner experto en Oracle y modernización enterprise
- Mostrar que la modernización no es solo migración, sino evolución a web + APIs + cloud + UX moderna
- Resaltar que usamos IA para acelerar assessment, documentación, extracción de lógica y reconstrucción

Público:
- Empresas con sistemas legacy Oracle Forms / Reports
- Gerentes de sistemas, CTOs, líderes de aplicaciones, responsables de modernización
- Organizaciones que quieren reducir dependencia de tecnología legacy sin poner en riesgo la operación

Estilo visual:
- Enterprise premium
- Fondo oscuro, acentos azules y violetas
- Secciones limpias, mucho aire, diseño moderno pero serio
- Imágenes tipo arquitectura, before/after, diagrams, app UI mockups y visuales de transformación
- Evitar stock photos de personas

Estructura:
1. Hero fuerte con headline, subheadline, CTA y visual “legacy → moderno”
2. Problemas del stack actual
3. Solución de Inthegra
4. Proceso de modernización en pasos
5. IA aplicada a la modernización
6. Before / after
7. Beneficios de negocio
8. Arquitectura objetivo
9. CTA final

Contenido clave:
- Modernización sin frenar la operación
- Oracle APEX, APIs/ORDS, OCI, integración y escalabilidad
- Reutilización de lógica de negocio
- Menor costo de mantenimiento
- Menor dependencia de especialistas legacy
- Base lista para analytics, integración y futuras capacidades de IA
